**Техническое Задание (ТЗ)**

**Веб-сайт торгового комплекса**

**1. Общие сведения**

**Название проекта:** Веб-сайт торгового комплекса (учебный MVP) **Тип проекта:** Полноценное веб-приложение с фронтендом и бэкендом **Среда запуска****:** Локально, Windows, браузер Google Chrome **Адаптивность****:** Не требуется **Цель****:** Разработать минимально жизнеспособный продукт сайта торгового комплекса с системой авторизации и корзиной

**2. Технический стек**

| **Часть** | **Технология** |
| --- | --- |
| Фронтенд | HTML5, CSS3, Vanilla JS |
| Бэкенд | Python 3.x + Flask |
| База данных | SQLite (через Flask-SQLAlchemy) |
| Авторизация | Flask-Session |
| Зависимости | requirements.txt |

**3. Цветовая палитра и стиль**

**Градиент фона** (слева направо):

- Левый край — Медовая роса: #F8F4FF

- Центр — Слоновая кость: #FFFFF0

- Правый край — Морская пена: #FFF5EE

css

																background: linear-gradient(to right, #F8F4FF, #FFFFF0, #FFF5EE);

**Дополнительные правила стиля:**

- Шрифт: чистый, читаемый (например, Inter или Segoe UI)

- Карточки магазинов/товаров: белый фон с лёгкой тенью

- Кнопки: тёмный акцентный цвет (например, #5C4A8F — тёмно-фиолетовый, гармонирует с медовой росой) либо на твой вкус

- Хедер и футер: слегка затемнённый, чтобы отделиться от фона

**5. Описание страниц**

**5.1 ****index.html**** — Главная страница**

**Назначение:** Витрина комплекса, точка входа

**Содержимое:**

- Хедер с логотипом, названием комплекса и навигацией

- Главный баннер (hero-секция) — название комплекса, короткий слоган, кнопка "Перейти к магазинам"

- Блок "О комплексе" — 2-3 предложения + иконки с преимуществами (парковка, фудкорт, акции)

- Превью-блок магазинов — 3-4 карточки со ссылкой "Смотреть все"

- Превью акций — 2-3 карточки со ссылкой "Все акции"

- Превью фудкорта — краткое описание + ссылка

- Футер

**5.2 ****shops.html**** — Каталог магазинов**

**Назначение:** Полный список магазинов с фильтрацией

**Содержимое:**

- Заголовок раздела

- Панель фильтров по категориям (кнопки): Все / Одежда / Электроника / Продукты / Красота / Другое

- Сетка карточек магазинов

**Карточка магазина содержит:**

- Логотип/фото магазина

- Название

- Категория

- Короткое описание

- Этаж и номер павильона

- Кнопка "Подробнее" (можно как заглушку)

**Логика фильтрации:** реализуется на JS — при нажатии на категорию карточки без нужного тега скрываются через display: none, остальные показываются. Данные магазинов хранятся прямо в JS-массиве (бэкенд не нужен для этого раздела).

**5.3 ****foodcourt.html**** — Фудкорт**

**Назначение:** Информация о заведениях питания

**Содержимое:**

- Заголовок и краткое описание фудкорта

- Карточки заведений (название, кухня, режим работы, фото)

- Опционально: фильтр по типу кухни (Азиатская / Европейская / Фастфуд)

**5.4 ****events.html**** — Акции и события**

**Назначение:** Актуальные акции магазинов и мероприятия

**Содержимое:**

- Карточки акций (название, магазин, описание, срок действия)

- Карточки событий (концерты, выставки, дни скидок) — можно статично

**5.5 ****contacts.html**** — Контакты**

**Назначение:** Информация о комплексе

**Содержимое:**

- Адрес, телефон, e-mail, режим работы

- Встроенная карта (Google Maps через <iframe>)

- Простая форма обратной связи (только фронтенд, без реальной отправки — можно сделать alert)

**5.6 ****login.html**** — Вход и регистрация**

**Назначение:** Авторизация пользователя

**Содержимое:**

- Две вкладки: "Войти" и "Зарегистрироваться" — переключаются через JS

- **Форма входа:** логин + пароль + кнопка

- **Форма регистрации:** имя пользователя + e-mail + пароль + подтверждение пароля + кнопка

- После успешного входа — редирект на cabinet.html

- Сообщения об ошибках (неверный пароль, пользователь уже существует)

**5.7 ****cabinet.html**** — Личный кабинет**

**Назначение:** Профиль пользователя и корзина

**Доступ****:** Только для авторизованных. Если пользователь не вошёл — редирект на login.html

**Содержимое:**

- Блок профиля: имя пользователя, e-mail (данные из БД)

- Кнопка "Выйти"

- Раздел "Корзина": 

- Список добавленных товаров (название, количество, цена)

- Кнопка "Удалить" у каждого товара

- Итоговая сумма

- Кнопка "Оформить заказ" — просто alert "Спасибо за заказ!" как заглушка

**6. Бэкенд — API маршруты**

**Авторизация**

| **Метод** | **Маршрут** | **Описание** |
| --- | --- | --- |
| POST | /register | Регистрация нового пользователя |
| POST | /login | Вход в систему, создание сессии |
| POST | /logout | Выход, удаление сессии |
| GET | /me | Вернуть данные текущего пользователя |

**Корзина**

| **Метод** | **Маршрут** | **Описание** |
| --- | --- | --- |
| GET | /cart | Получить список товаров в корзине |
| POST | /cart/add | Добавить товар |
| DELETE | /cart/remove/<id> | Удалить товар |

| **Поле** | **Тип** | **Описание** |
| --- | --- | --- |
| id | INTEGER | Первичный ключ |

| quantity | INTEGER | Количество |

**8. Общие компоненты (хедер и футер)**

Так как это чистый HTML без фреймворков, хедер и футер **дублируются** в каждом файле. Это нормально для учебного проекта. Стили выносятся в header.css и footer.css.

**Хедер содержит:**

- Логотип + название комплекса (слева)

- Навигация: Главная / Магазины / Фудкорт / Акции / Контакты (по центру или справа)

- Кнопка "Войти" или иконка кабинета (если вошёл) — справа

**Футер содержит:**

- Название комплекса

- Краткие ссылки навигации

- Контактная информация

- Копирайт
